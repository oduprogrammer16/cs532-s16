I am absolutely in love with Laravel 4, but I guess that shouldn&#8217;t come as a surprise looking at the fact that it is just the most awesome framework there is!</p><p>If you love Laravel as much as I do, there is no doubt you want to take it for a spin and realize its full potential once you have the basics down.</p><p>One of the beautiful features of Laravel are the easy-to-install, easy-to-use Laravel 4 packages that you can include in your Laravel project to help you start up your next project as quickly as possible.</p><p>Without further ado, here are some of the packages &#8211; in no particular order &#8211; that I have been using a lot to make coding in Laravel even easier and faster than it already is.</p><h3>Laravel 4 Generators &#8211; Jeffrey Way</h3><p>Hands down <em>the</em> most useful packages out there to be used in every Laravel 4 project.</p><p>It pretty much does what it says on the box. It generates code for you, and does this in such a way that it makes CodeIgniter&#8217;s 20-minute-blog tutorial pale in comparison. 20 minutes? With Laravel 4 generators you can do it in about 2 minutes flat!</p><p>It can automagically generate the following with a few simple artisan commands:</p><ul><li>Models</li><li>Views</li><li>Controllers</li><li>Seeds</li><li>Migrations</li><li>Forms</li><li>Tests</li><li>Resources (this will generate a model, migration and controllers for you)</li><li>Scaffolds (now this will generate pretty much everything to get you started prototyping your application at lightning speed)</li><li>Pivot tables</li></ul><p>In addition to this, there is also a <a href="http://net.tutsplus.com/tutorials/tools-and-tips/pro-workflow-in-laravel-and-sublime-text/" target="_blank">Sublime Text plugin</a> available that will give you access to all these generators from within Sublime Text.</p><p><strong>Github: </strong><a href="https://github.com/JeffreyWay/Laravel-4-Generators" target="_blank">https://github.com/JeffreyWay/Laravel-4-Generators<br /> </a><strong></strong></p><h3>Sentry &#8211; Cartalyst</h3><p>By default, Laravel 4 comes with a basic User model which includes password reminders and other basics. Cartalyst however will offer you a full user system with its amazing <strong>Sentry</strong> package.</p><p>It has the following features:</p><ul><li>Configurable authentication (can use any type of authentication required, such as username or email)</li><li>Authorization</li><li>Activation of user <em>(optional)</em></li><li>Groups and group permissions</li><li>&#8220;Remember me&#8221;</li><li>User suspension</li><li>Login throttling <em>(optional)</em></li><li>User banning</li><li>Password resetting</li><li>User data</li><li>Interface driven &#8211; switch out your own implementations at will</li></ul><p>Not only is it available in Laravel 4, but also in FuelPHP1, CodeIgniter 3 and native PHP (through composer). It currently sits at version 2.1 and has an alpha of version 3. If you are looking to make a website or application that will have to handle multiple users, and multiple types of users (Administrator, normal user, premium user, &#8230;) then Cartalyst Sentry is definitely the way to go!</p> <!-- Quick Adsense WordPress Plugin: http://quicksense.net/ --><div style="float:none;margin:0px 0 0px 0;text-align:center;"> <br /><script async src="//pagead2.googlesyndication.com/pagead/js/adsbygoogle.js"></script> <!-- Responsief 2 --> <ins class="adsbygoogle"

I personally stopped using this a while ago but if you don&#8217;t want to dive into the Laravel 4 documentation to look up the naming of every single function you should definitely give laravel-ide-helper by barryvdh a try.</p><p>To use this package, you can either grab a pre-generated version and drop it in your project, or get the full package and generate your own autocompletion file.</p><p>It has automatic phpDoc generation for Laravel Facades, and for Models.</p><p><strong>Github: </strong><a href="https://github.com/barryvdh/laravel-ide-helper" target="_blank">https://github.com/barryvdh/laravel-ide-helper</a></p><h3>Laravel Test Helpers &#8211; Jeffrey Way</h3><p>Another great package from Laravel demi-god: Jeffrey Way</p><p>This package eases the process of writing tests for a Laravel application by offering:</p><ul><li>A Factory utility (quickly create and populate models)</li><li>Model test helpers (<code>assertValid</code>, <code>assertBelongsTo</code>, etc.)</li><li>Controller test helpers (<code>assertSee</code>)</li><li><code>Assert</code> and <code>Should</code> PHPUnit wrappers</li></ul><p>Testing in Laravel is very important and should not be overlooked if you want your application to be as solid as possible.</p><p>There isn&#8217;t much else to be said about this package other than that I highly recommend it if you would like to make testing in Laravel just that little bit easier than it already is!</p><h3>Ardent &#8211; laravelbook</h3><p>Self-validating smart models for Laravel. Validation is something you will without doubt use in every single Laravel project. If you have ever worked with CakePHP, you know how handy this feature is. You can write your model and specify all the validation that needs to be done before saving your model to your database. Then when you save the Model inside of your Controller, validation will happen automatically instead of cluttering your Controllers with a lot of redundant code.</p><p>Using Ardent for Laravel you can specify all the validation, error messages, &#8230; directly inside your model (valid e-mail, username, &#8230;)</p><p>I use this package in almost every project I start up and it cuts down my development time by a great amount of time!</p><p><strong>Github:</strong> <a href="https://github.com/laravelbook/ardent" target="_blank">https://github.com/laravelbook/ardent</a></p><p>I hope this post will help everyone that is looking to speed up their development time with some packages that can be used in almost every project.</p><p>If you have any suggestions, or packages that you use in a lot of your projects over and over again, feel free to leave a commend below and I&#8217;ll update this post with some information on the package.</p><div
